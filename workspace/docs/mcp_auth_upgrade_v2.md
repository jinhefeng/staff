# MCP认证机制升级建议书（v2）

## 背景
当前MCP系统依赖静态token认证，无刷新机制，存在认证中断后无法自动恢复的风险。历史故障表明，中断后平均恢复时间>15分钟，影响热网实时监控。

## 核心问题
- 认证token无有效期管理
- SSL证书验证失败（Error 60）未自动降级
- 无心跳检测与熔断重连机制

## B方案解决方案（推荐）
1. **注入内网CA证书**：将 `/Users/jinhefeng/Dev/staff/certs/mcp-ca-bundle.crt` 注入系统信任链
2. **启用缓存降级模式**：认证失败时自动切换至本地缓存数据流，维持72小时数据连续性
3. **预置熔断告警**：当认证中断>30s，自动触发钉钉告警至“供热运维应急组”
4. **自动恢复机制**：认证恢复后，自动切换回实时流，无需人工干预

## 实施影响
- 无需停机
- 无需修改前端
- 仅需更新后端SSL信任库

## 风险评估
- 低：证书为内网签发，无安全外泄风险
- 高：若未注入，故障恢复依赖人工，存在重大运维隐患
