# 🚀 Staff “言出必行” 防敷衍机制修复总结

本次迭代解决了 Staff 在面对由于客观原因失效的技能或长耗时任务时，**只口头承诺而不实际执行动作**的“嘴炮”现象。

## 🎯 修复目标
确保系统能够 100% 捕获并拦截模型产生的“空头支票”。只要模型在回复中隐含了“稍后处理”、“去寻找”等意图，系统就能强制执行工单创建，保证任务可追踪。

## 🛠️ 关键改动 (What changed)

1. **增强系统规则指引 (`AGENTS.md`, `context.py`)**：
   - 补充了第 8 条“承诺必达与防敷衍机制 (Anti-Lip-Service Protocol)”。
   - 强化了对 `BACKGROUND TASKING` 的要求，明确指出如果做出口头承诺但不调用工具，将会被底层程序接管甚至扣减分数。

2. **新增基于 LLM 的意图审查器 (`sanitizer.py`)**：
   - 在 `SanitizerAgent` 中新增了 `check_promise_intent` 异步方法。
   - 这相当于一个**专门负责鉴谎和意图审核的小模型/独立调用**。它会审查即将发送出去给用户的整段文本，若检测到承诺“以后弄、查一查”，便返回 `PROMISE: True`。

3. **核心拦截与物理兜底逻辑 (`loop.py`)**：
   - 在主循环发送 `OutboundMessage` 前嵌入拦截卡口。
   - **触发条件**：当前迭代无任何功能性 `tools_used`，且开启了拦截器验证。
   - **兜底动作**：一旦被鉴定为嘴炮，系统会在给用户的回复尾部强插通告 `【系统拦截】：检测到口头承诺但未立刻执行动作。已自动挂载后台工单，将在闲暇时异步处理。`，并自动调用 `TicketManager` 创建系统溯源工单。

## 🧪 验证与测试
- 编写了单体测试脚本验证 `check_promise_intent` 可以准确在模拟语句中捕获到 `PROMISE` 状态。
- **用户联调测试**：请 Master (用户) 亲自在钉钉/其他客户端对 Staff 发布一条类似“技能坏了，帮我找个新接口重新搞个”的指令。期待能看到 Staff 触发系统拦截并自动建单的效果。
