# Staff 钉钉原生企业机器人对接教程

果断放弃不稳定的非官方微信大底座，拥抱**官方、免费且绝对稳定**的钉钉原生大模型机器人，绝对是一个无比明智的架构师级决策！

钉钉官方为企业内部机器人提供了长连接（Stream 模式）和 Webhook 模式，Nanobot 都做了原生完美适配。既然您的服务器上配有 Caddy 反代引擎，下面这套规范将让您的 Staff 以专属 AI 同事的身份降临钉钉！

---

## 步骤一：创建钉钉应用

1. 使用您的电脑浏览器登录 [钉钉开发者后台](https://open-dev.dingtalk.com/)。
2. 确保页面的左上角切换到了您的**目标企业/团队**。
3. 点击顶部导航栏的 **[应用开发]** -> **[企业内部开发]** -> **[机器人]**。
4. 点击右上角的 **[创建应用]**：
   - **应用名称**：填入心仪的名字（如：Staff 或智能助理）
   - **应用描述**：随便写点介绍。
   - **应用图标**：可以上传一张带感的头像。
   - 点击保存完成创建。

## 步骤二：获取 API 密钥

创建好后，停留在该应用的**[凭证与基础信息]**页面：
- 您会看到一组非常关键的凭证：
  1. `AppKey` (或是 Client ID)
  2. `AppSecret` (或是 Client Secret)

> [!IMPORTANT]
> 请复制这两串秘钥，然后打开您 Macbook 本地工程里的 `config.json`，在 `dingtalk` 的配置块里填入它们：
> ```json
>     "dingtalk": {
>       "enabled": true,
>       "clientId": "替换成您刚刚复制的 AppKey",
>       "clientSecret": "替换成您刚刚复制的 AppSecret",
>       "allowFrom": []
>     }
> ```

## 步骤三：开通机器人的接受消息权限

Nanobot 的 DingTalk 模块默认使用安全且穿透极佳的 **Stream (流式) 模式**。
这意味着您甚至不需要 Caddy，只要您的机器能上网，钉钉官方就会把消息推送给本地或者云端的脚本！

1. 在左侧菜单栏找到 **[权限管理]**。
2. 搜索并勾选以下权限（点击申请）：
   - `企业内机器人发送消息权限`
3. 回到左侧菜单，点击 **[机器人与消息推送]**。
4. 开启机器人功能，完善信息后，**将消息接收模式设置为 [Stream 模式]**。

## 步骤四：发布并拉它进群

1. 在左侧点击 **[版本管理与发布]**。
2. 点击右上角的创建新版本，选好可用范围（比如只给您自己用，或者全公司用），点击发布！
3. 现在，打开桌面版或手机版的钉钉客户端，搜索您刚才起的机器人名字（Staff）。

> 💡 **起飞准备**：
> 在您本地开一个终端，运行 `./start.sh` 点火！
> 当控制台出现翠绿色的 `[✓] Dingtalk module active` 神谕时，在钉钉里给它发一句“你好呀”，它就会带着满血架构师的算力回应您啦！
