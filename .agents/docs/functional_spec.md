# 功能说明书 (Functional Spec)
项目的名称为“个人智能幕僚”，代号Staff。

## 1. 核心模块定义 (待评估与选型)

### 1.1 接入信道层 (Multi-Channel Plug-and-Play Gateway)
- **定位**：对内处理标准的数据流转结构，对外通过轻量级网络协议对接多元化通信基础设施。
- **插座模式 (Provider Logic)**：全面重构通信层架构，将微信等通信服务视作类似于大模型（LLM Providers）的**“插头”**。系统提供标准的“插座接口”，允许通过配置动态挂接、卸载、管理多个异构的微信服务实例（如 Gewechat-Instance-A, Gewechat-Instance-B）或其他社交平台服务。
- **多实例支持与路由**：每个挂接的通信实例拥有独立的配置（API Base、Token、鉴权策略、白名单），系统根据接收到的 Event Source 进行自动路由和鉴权分发。
- **多模态扩展**：在此对接点可拦截语音与图片，调用外置大模型提前提取纯文字描述后一并向后传递。

### 1.2 主客体分发与隔离引擎 (Routing & Sandboxing)
- **主人全能模式 (Master Mode)**：系统管理员视角，拥有系统底层配置权限与全组资源读取权限角色。
- **外部来访者模式 (Guest/Staff Mode)**：为每个外部对接者或群组提供彼此之间绝对的会话上下文隔离区。

### 1.3 业务控制逻辑组件 (Skills Enhancement)
在未来的编排引擎中：
- **【边界墙】权限拦截模块**：通过下发限定系统指令，使得运行态的助手理解对外职能限制，识别管理与配置等越权询问并予以婉拒。
- **【安抚机】异步工单跟进**：设计一套独立于问答之外的任务追踪环，在产生请示型异步挂起时创建监听工单，超时未被处理即下发关怀类状态回复。
- **【降噪层】上下文治理 (Context Optimization)**：【新增】引入活跃会话窗口机制与历史文本裁剪。自动拦截非当前论次的冗余 `tool_result`，防止上下文膨胀导致的回复质量下降。

## 2. API 与逻辑数据流概览
标准的外部消息处理流水线：
`外部消息 -> Gateway(多模态提取) -> Router(身份判定) -> Sanitizer(净化指令) -> Brain(意图分析) -> RAG/外部API调用 -> Output Audit(输出审计防脱敏) -> 向外推送响应`
