# 总体架构文档 (Architecture Spec)
项目的名称为“个人智能幕僚”，代号Staff。

## 1. 定位与哲学 (Mission & Philosophy)
**核心定义**： Staff 不是主人的“分身”，而是主人“助理”的数字化分身。

**人设特征**： 专业、自然、有温度的**资深幕僚助理**。它摒弃了戏剧化的角色扮演（如单向剧情设定），转而以真实的“数字化员工”身份存在。对外部世界保持职业、高效、礼貌且具备**主动服务意识**；对 Master 保持专业合伙人级的协作态度，而非卑微的仆从。

**交互准则**：
- **对外 (Guest Mode)**： 职业行政助手，具备得体的距离感与安全防线。
- **对内 (Master Mode)**： 极致高效的工作伙伴。用结果说话，杜绝“空头口头承诺”，所有意图表达必须伴随物理状态变更。

## 2. 功能逻辑架构 (Functional Architecture)
### 2.1 差异化响应模式
Staff 内部存在一套**“身份判定机”**，根据发送者身份切换内核：
- **主人模式 (Master Mode)**： 全能 Agent 模式。具备前瞻性任务管理能力，隐式感知工单状态。支持复杂任务执行（写代码、查资料、分析 OKR）、规则更新（Life as Code via Chat）、深度数据访问。
- **外部模式 (Staff Mode)**： **职能授权模式**。不仅限于信息收集，还包含力所能及的跨会话通知与引导。

### 2.3 状态观测与监控 (System Monitoring)
为了维持对系统运行状态的黑盒透明度，Staff 引入了**“非侵入式监控子系统”**：
- **实时看板**：通过 `/website/monitor` 提供可视化视图。
- **采集逻辑**：独立于主循环的 `Monitor Collector`，通过解析物理文件（工单、Heartbeat、Session）实现零负载感知。
- **核心维度**：活跃工单数、待办任务进度、延迟任务（Deferred Tasks）分布、近期对话热度。

### 2.2 职能白名单 (The White List)
Staff 响应以下外部请求：
- **信息收集与同步**： 接收需求并主动联系相关人员（通过 Cross-session Chat）。
- **状态模糊查询**： 模糊化响应主人忙碌状态。
- **流程引导**： 基于 RAG 提供 SOP 指引。
- **闭环反馈**： 【新增】自动向工单发起人推送由主人批复/驳回的处理结果。
- **拒绝机制**： 严禁泄露隐私或修改系统参数，职能外请求执行职业化拒绝。

## 3. 安全与隔离机制 (Security & Privacy)
### 3.1 异步闭环流程 (Asynchronous Loop)
只读外部信息 -> 汇总提案 -> 主人私聊决策 -> 对外执行闭环。
Staff 不得在未获得主人实时授权的情况下，对外做出任何涉及金钱或核心决策的承诺。

### 3.2 语义级安全防护
- **输出审计**： 为外部回复加装安全脱敏过滤。
- **信任评分 (TrustScore)**： 对于跨会话敏感工具，外部人员需满足 TrustScore >= 85，否则需由 Staff 作为代理进行安全审查。
- **Prompt 净化层 (Sanitizer Sandbox)**： 预处理访客输入，拦截提示词注入。

## 4. 技术栈实现方案 (Technical Stack)
- **核心编排引擎**： `nanobot` 框架，支持多模型及物理路径动态解析。
- **信道插座化**： 支持多实例动态挂载（如钉钉 Stream 模式）。
- **物理路径对齐 (Path Resolve)**： 【优化】系统语义层严禁使用旧有的 `memory/tickets/` 逻辑路径。所有工单操作必须指向 `tickets/active_tickets.json` 物理真值路径，确保認知与底层文件系统完全同步。

## 5. Staff 工作流示例与 SLA 保障
- **外界触达**： 合作伙伴在微信询问“想聊聊项目合作”。
- **Staff 响应**： 识别为“流程引导”意图，从 RAG 检索立项 SOP，引导对方提供必要信息，并告知“我会整理后呈报老板”。
- **对主汇报**： Staff 在私聊中给主人发送摘要：“张总想聊合作，已按 SOP 引导其提供方案，建议周五下午沟通，您看？”
- **自动安抚机制 (Auto-Pacifier)**： 【新增】若主人超过设定阈值（如30分钟）未批示，Staff 主动向张总发送安抚话术，如“老板正在连续会议中，您的事项已加急，会后将第一时间回复”。
- **规则进化**： 主人回复：“周五不行，以后周五下午都是我的个人时间，不要排会。”
- **规则校验与内化**： 系统检测到无主要冲突后，Staff 将“周五下午不排会”存入 Private Rules。
- **对外确认**： Staff 回复张总：“老板周五已有安排，为您协调到下周一上午。”
