# 总体架构文档 (Architecture Spec)
项目的名称为“个人智能幕僚”，代号Staff。

## 1. 定位与哲学 (Mission & Philosophy)
**核心定义**： Staff 不是主人的“分身”，而是主人“助理”的数字化分身。

**人设特征**： 具备独立社交人格（如化名、独立微信账号）的数字幕僚。它对外部世界保持职业、高效、有距离感的形象；对主人保持绝对忠诚、全能且可受训。

**交互准则**：
- **对外**： 拟人化交互，消除“机器人感”，作为社交第一道防火墙。
- **对内**： 纯聊天式受训。主人通过对话输入经验与规则，Staff 实现自我进化。

## 2. 功能逻辑架构 (Functional Architecture)
### 2.1 差异化响应模式
Staff 内部存在一套**“身份判定机”**，根据发送者身份切换内核：
- **主人模式 (Master Mode)**： 全能 Agent 模式。支持复杂任务执行（写代码、查资料、分析 OKR）、规则更新（Life as Code via Chat）、深度数据访问。
- **外部模式 (Staff Mode)**： 职能受限模式。仅限社交对接、信息采集、流程引导。

### 2.2 职能白名单 (The White List)
为了防止滥用，Staff 仅响应以下外部请求：
- **信息收集**： 接收并结构化外部需求。
- **状态查询**： 模糊化响应主人忙碌状态。
- **流程引导**： 基于 RAG 提供立项、法务、财务等 SOP 指引。
- **事务预约**： 协调时间与同步参与人。
- **拒绝机制**： 职能范围外的请求（如代写代码），由 Staff 进行婉转的职业化拒绝。

## 3. 安全与隔离机制 (Security & Privacy)
### 3.1 异步闭环流程 (Asynchronous Loop)
为了确保决策权始终在主人手中，Staff 遵循：只读外部信息 -> 汇总提案 -> 主人私聊决策 -> 对外执行闭环。
Staff 不得在未获得主人实时授权的情况下，对外做出任何确定性承诺（涉及时间、金钱、决策）。

### 3.2 语义级安全防护
- **输出审计**： 在回复外部前，通过安全规则层进行再加工，强制脱敏（如将具体会议内容模糊化为“Busy”）。
- **权限分级**： 外部访问权限仅限于“能力插件”的输出结果，而非原始数据库。
- **Prompt 净化层 (Sanitizer Sandbox)**： 【新增】在 Staff Mode 下，前置拦截并丢弃任何带有“管理、配置、查看系统规则”意图的提示词输入，防范提示词注入攻击。

## 4. 技术栈实现方案 (Technical Stack)
- **核心编排引擎 (To Be Determined)**：原定方案 (NanoClawd) 因模型兼容性问题已废弃，目前正在重新进行框架选型，需原生支持多模型及 Gemini API 接入。
- **接入信道层 (Multi-Channel Plug-and-Play Gateway)**：
  - 【插座化架构升级】全面升级信道系统的解耦机制，引入“插座模式 (Provider Logic)”。网关从“单一实例硬编码”转向“多实例动态挂载”。
  - 【异地分布式部署 (Distributed Deployment)】：【新增】遵循 API Gateway 模式，微信大底座 (Gewechat 容器) 与主核心编排程序 (Staff CLI) 在物理架构上可**完全剥离**。Gewechat 作为无状态的独立公网或局域网纯净服务运行（原生镜像，随开随关）。双方仅通过 RESTful API (本地出站) 与 Webhook (反向隧道穿透入站) 互致消息。
  - 【入站拦截与鉴权体系 (Webhook & Auth)】：微信（如 Gewechat）作为独立的“第三方连接器（Provider）”运行。Staff 核心在初始化该 Provider 时，不仅需要按需验证其 Token 凭证（懒加载），还必须**自动化地调用 Provider 接口，注册将事件反向推送到 Staff 本地 Webhook 端点**（本地开发时需借助 ngrok 等隧道暴露给云端 Gewechat），以实现真正的双向通信收发握手与接口防篡改鉴权。
- **沙箱与技能体系 (Sandboxing & Skills)**：
  - 需借助新选型框架的能力，重新设计物理隔离或逻辑隔离边界（如 Docker 或 LangChain 内存隔离），实现主客体权限控制（Boundary Control）及延时触法（Timeout Pacifier）。

## 5. Staff 工作流示例与 SLA 保障
- **外界触达**： 合作伙伴在微信询问“想聊聊项目合作”。
- **Staff 响应**： 识别为“流程引导”意图，从 RAG 检索立项 SOP，引导对方提供必要信息，并告知“我会整理后呈报老板”。
- **对主汇报**： Staff 在私聊中给主人发送摘要：“张总想聊合作，已按 SOP 引导其提供方案，建议周五下午沟通，您看？”
- **自动安抚机制 (Auto-Pacifier)**： 【新增】若主人超过设定阈值（如30分钟）未批示，Staff 主动向张总发送安抚话术，如“老板正在连续会议中，您的事项已加急，会后将第一时间回复”。
- **规则进化**： 主人回复：“周五不行，以后周五下午都是我的个人时间，不要排会。”
- **规则校验与内化**： 系统检测到无主要冲突后，Staff 将“周五下午不排会”存入 Private Rules。
- **对外确认**： Staff 回复张总：“老板周五已有安排，为您协调到下周一上午。”
