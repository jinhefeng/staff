# DDD 协作开发流程规范

## 1. 方案先行 (Mandatory)
在修改任何核心逻辑前，Agent 必须以以下结构输出实施方案：
- 【变更背景】：简单描述为什么要改。
- 【模型调整】：涉及的聚合根、实体、值对象的属性或关系变化。
- 【逻辑规则】：业务逻辑的核心变动。
- 【受影响文档】：列出预计需要更新的 .agents/docs 文件。

## 2. 确认机制
- Agent 必须停止操作并询问：“以上方案是否符合您的设计意图？”
- 必须在获取用户明确的“确认”或针对方案的“批注/修正”后，才能进入下一步。

## 3. 文档同步触发
- 在用户确认方案后，Agent 必须立即调用 `Extract-Design-Change` Workflow。
- 流程：方案确认 -> 调用 Workflow 更新文档 -> 展示文档 Diff -> 开始编写代码。
