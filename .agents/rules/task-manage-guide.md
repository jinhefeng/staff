1. **核心任务文件与层级结构**：artifacts目录下的`task.md`是我们最关键的任务指导书，是用户与 Agent 协作的唯一“目标追踪系统”。它必须拥有清晰的父子层级结构，禁止流水账似的单层清单。
2. **任务建档优先（Task First）原则**：无论何时识别到用户的**新目标、新需求或突发方向调整**，都必须将其分解为具体的任务流条目。此时需立即打开 `task.md` 写入这些带有层级的待办事项（可标记状态例如针对新需求 `[Needs Review]`）。
3. **拦截确认流（Blocker Check）**：**【强制截点】**在 `task.md` 中增加或修改了任务条目后，在正式编写代码或输出长篇方案之前，必须先将修改后的任务列表向用户展示，并明确请求用户确认。只有获得明确的授权（如 LGTM）后，方可启动对应任务的开发流。
4. **历史持久化原则（History Persistence）**：**【强制准则】**严禁以任何理由（如“为了简洁”）删除或精简 `task.md` 中已完成的任务记录。历史记录必须全量保留以作为项目进展的审计依据。
5. **实时更新（Update Intime）**：artifacts目录下的`task.md`需要与.agents/docs/task.md保持同步。
